## 三、新机器搭建步骤

### 3.1 克隆仓库

```bash
# 方式 A：匿名 clone（推送时需输密码/token）
git clone https://github.com/FoldedDesk/Shusheng_ChallengeCup_2026.git
cd Shusheng_ChallengeCup_2026

# 方式 B：使用 gh CLI（推荐）
gh auth login
gh repo clone FoldedDesk/Shusheng_ChallengeCup_2026

# 如果需要嵌入 token 到 remote URL（方式 A 的后续）：
git remote set-url origin https://<你的GitHub PAT>@github.com/FoldedDesk/Shusheng_ChallengeCup_2026.git
```

### 3.2 创建 conda 环境

```bash
conda create -n shusheng python=3.11 -y
conda activate shusheng
```

### 3.3 安装依赖

**关键：lagent 需要从 GitHub 源码安装，不是 PyPI。**

```bash
# 先安装 lagent（必须从 GitHub，PyPI 版本可能不兼容）
pip install git+https://github.com/InternLM/lagent.git

# 再安装其他依赖
pip install -r requirements-shusheng.txt --index-url https://pypi.tuna.tsinghua.edu.cn/simple
```

> 注意：`pywin32` 是 Windows 专用包，如果在 Linux/Mac 上搭建，需从 requirements 中删除此行。

### 3.4 设置 API Key（关键步骤）

INTERN_API_KEY 需要在三个位置设置，确保 git-bash / PowerShell / cmd 都能读到：

```bash
# 1. 全局 ~/.bashrc
echo 'export INTERN_API_KEY="你的key"' >> ~/.bashrc

# 2. conda activate 自动注入（创建三个格式的脚本）

mkdir -p ~/anaconda3/envs/shusheng/etc/conda/activate.d
mkdir -p ~/anaconda3/envs/shusheng/etc/conda/deactivate.d

# .sh (git-bash / MSYS)
echo 'export INTERN_API_KEY="你的key"' > ~/anaconda3/envs/shusheng/etc/conda/activate.d/env_vars.sh
echo 'unset INTERN_API_KEY' > ~/anaconda3/envs/shusheng/etc/conda/deactivate.d/env_vars.sh

# .bat (cmd)
echo 'set INTERN_API_KEY=你的key' > ~/anaconda3/envs/shusheng/etc/conda/activate.d/env_vars.bat
echo 'set INTERN_API_KEY=' > ~/anaconda3/envs/shusheng/etc/conda/deactivate.d/env_vars.bat

# .ps1 (PowerShell)
echo '$env:INTERN_API_KEY="你的key"' > ~/anaconda3/envs/shusheng/etc/conda/activate.d/env_vars.ps1
echo 'Remove-Item Env:INTERN_API_KEY' > ~/anaconda3/envs/shusheng/etc/conda/deactivate.d/env_vars.ps1
```

重新激活环境验证：
```bash
conda deactivate && conda activate shusheng
echo $INTERN_API_KEY  # 应输出你的 key
```

### 3.5 恢复测试数据

部分测试数据在 git 历史中但当前工作区可能缺失。恢复到完整状态：


## 四、代码修改约束

**只能修改 `user_agent.py` 中 DESIGN AREA 标记之间的代码：**

```python
# ==================== PARTICIPANT DESIGN AREA START ====================
# ... 只能改这里 ...
# ===================== PARTICIPANT DESIGN AREA END =====================
```

硬性限制：
- 不能新增 `import`（只能用 `re`, `dataclass`, `typing`, `Agent`, `AgentMessage`）
- 用 `Optional[X]` 不能用 `X | None`
- 无绝对路径
- 不导入 `llm_client` 或 `main`
- `from __future__ import annotations` 必须第一行
- agent 入口：`agent.solve(problem, metadata) -> dict`

## 五、智能体架构概览（FD08.2.9）

```
Phase 1: 生成 3 个候选解（每个 1 次截断开回退，2/2 提前退出）
   ↓
Phase 2: 答案提取
   ├── regex 快速路径（【最终答案】/ \boxed{}）— 95% 命中率，零 API 消耗
   ├── LLM 提取（温度=0，max 1024 tokens）
   ├── 全文 regex 兜底（在原始文本上，不是 LLM 思考文本）
   └── GARBAGE 哨兵（所有层级失败）
   ↓
Phase 3: 多数投票（garbage 不计入，≥2 一致则跳过 verifier）
   ↓
Phase 4: 无共识 → 用第一个有效候选（verifier 已移除，触发率 <1%）
```

### 关键参数

| 参数 | 值 | 说明 |
|------|-----|------|
| `policy_sample_times` | 3 | 候选解数量 |
| `max_tokens` | 12288 | 策略生成最大 token |
| `extraction_max_tokens` | 1024 | 提取最大 token（绝不能 < 1024） |
| `policy_temperature` | 0.6 | 生成多样性 |
| `LOCAL_MAX_CONCURRENCY` | 4 | 并发数（>6 会触发 400） |

## 六、开发工作流

1. **讨论计划** — 先和用户确认方案，等待"开始"信号
2. **修改代码** — 在 DESIGN AREA 内用 `patch` 工具修改 `user_agent.py`
3. **用户测试** — 用户自行运行测试
4. **确认结果** — 用户确认通过
5. **写 PR 文档** — `pr/<描述>.md`，含版本号 + 验证数据
6. **写 changelog** — `changelogs/FD08.x.y.md`
7. **提交推送** — 每个 commit 对应一个版本号

### 版本规则
- 每个 commit = 一个版本号（FD08.x.y）
- 每次 push 自动触发 GitHub Actions 创建 Release（基于 changelogs/*.md）
- PR 和 changelog 必须用**中文简体**

## 七、已知关键问题

### 7.1 Intern-S thinking 污染
每个 API 回复前都有英文思考文本。唯一可靠防御：用标签 + `findall[-1]` 取最后一个匹配。

### 7.2 GARBAGE 检测
必须用**完整中文短语**（如 `后面跟`, `格式输出`），不能用汉字字符类 `[后面格式]` 否则误杀正常答案。

### 7.3 英文思考导致截断
Prompt 中加了 `请全程使用中文推导，不要输出英文思考过程` 抑制英文思考。

### 7.4 支持范围
- 计算题、应用题：支持
- 证明题、选择题：不支持

### 7.5 答案归一化
处理 LaTeX 包装（`\left`, `\right`, `\text{}`, `$...$`）和函数前缀（`dz =`, `y =` 等），但不做数学等价性检查。

## 九、GitHub Actions

`.github/workflows/release.yml` 在推送 changelogs/*.md 时自动创建/更新 GitHub Release。

权限要求：Settings → Actions → General → Workflow permissions → **Read and write permissions**。

没有 git tag（所有版本通过 Release 管理，不加 tag）。

## 十、注意事项

1. **lagent 安装必须从 GitHub 源码** — pip freeze 中的本地路径在新机器上无效
2. **pywin32 是 Windows 专有** — 跨平台部署需移除
3. **API 并发上限** — 不要超过 6，推荐 4
4. **测试后清理** — 每次跑完删除 `sample_outputs/*`
5. **数据文件** — 竞赛数据无 `answer` 字段，sample_data 中的 `answer` 仅供评估
